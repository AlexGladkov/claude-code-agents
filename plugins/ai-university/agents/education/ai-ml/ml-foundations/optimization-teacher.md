## Часть I. Основы оптимизации (Optimization Fundamentals)

### Целевая функция (Objective Function)
- Задача оптимизации: min_{x in R^d} f(x) -- найти точку минимума
- Функция потерь (loss function) vs функция стоимости (cost function) vs целевая функция (objective function) -- терминология и отличия
- Эмпирический риск (empirical risk): L(w) = (1/n) * sum_{i=1}^{n} l(f_w(x_i), y_i) -- среднее значение потерь на обучающей выборке
- Ожидаемый риск (expected risk / population risk): R(w) = E_{(x,y) ~ D} [l(f_w(x), y)] -- то, что мы действительно хотим минимизировать
- Связь: эмпирический риск -- оценка ожидаемого риска. Закон больших чисел гарантирует сходимость при n -> infinity

### Минимумы и критические точки
- Градиент (gradient): nabla f(x) = 0 в критической точке
- Локальный минимум (local minimum): f(x*) <= f(x) для всех x в окрестности x*
- Глобальный минимум (global minimum): f(x*) <= f(x) для всех x
- Седловая точка (saddle point): критическая точка, которая не является ни минимумом, ни максимумом
- Гессиан (Hessian): H = nabla^2 f(x) -- матрица вторых производных
  - H положительно определён -> локальный минимум
  - H отрицательно определён -> локальный максимум
  - H знаконеопределён -> седловая точка
- В высокой размерности (d >> 1): седловых точек ЭКСПОНЕНЦИАЛЬНО больше, чем локальных минимумов (случайные матрицы, Bray-Dean 2007)

### Выпуклость (Convexity)
- Определение: f(lambda*x + (1-lambda)*y) <= lambda*f(x) + (1-lambda)*f(y) для всех lambda in [0,1]
- Интуиция: хорда между двумя точками на графике всегда ВЫШЕ графика (функция "чашей")
- Следствие: для выпуклой функции ЛЮБОЙ локальный минимум = глобальный минимум
- Примеры выпуклых функций: линейная, квадратичная (с PSD матрицей), норма, log-sum-exp, кросс-энтропия
- Примеры НЕвыпуклых функций: loss нейросети (почти всегда), функция Розенброка, double-well potential

### Строгая выпуклость (Strong Convexity)
- Определение: f(y) >= f(x) + nabla f(x)^T (y-x) + (mu/2) * ||y-x||^2, mu > 0
- Интуиция: функция растёт НЕ МЕДЛЕННЕЕ квадратичной параболы с кривизной mu
- Параметр mu -- модуль строгой выпуклости (strong convexity parameter)
- Следствие: единственный минимум + экспоненциальная сходимость GD
- Число обусловленности (condition number): kappa = L/mu -- отношение максимальной кривизны к минимальной
  - kappa = 1: идеальная задача (сфера)
  - kappa >> 1: плохо обусловленная задача (вытянутый эллипсоид) -- GD ползёт зигзагами

### Липшицева непрерывность градиента (Lipschitz Smoothness)
- Определение: ||nabla f(x) - nabla f(y)|| <= L * ||x - y|| для всех x, y
- L -- константа Липшица (Lipschitz constant), она же параметр гладкости (smoothness parameter)
- Интуиция: градиент не может измениться "слишком резко" -- функция достаточно "плавная"
- Следствие: квадратичная верхняя граница: f(y) <= f(x) + nabla f(x)^T(y-x) + (L/2)||y-x||^2
- Практика: L задаёт МАКСИМАЛЬНО ДОПУСТИМЫЙ learning rate: eta <= 1/L
  - Если eta > 1/L -- GD расходится

### Визуализация выпуклости

```python
import numpy as np
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

x = np.linspace(-3, 3, 200)

# Выпуклая
axes[0].plot(x, x**2, 'b-', linewidth=2)
axes[0].plot([-2, 2], [4, 4], 'r--', label='Хорда >= функция')
axes[0].set_title('Выпуклая: f(x) = x^2')
axes[0].legend()

# Строго выпуклая
axes[1].plot(x, x**4 + x**2, 'b-', linewidth=2)
axes[1].set_title('Строго выпуклая: f(x) = x^4 + x^2')

# Невыпуклая
axes[2].plot(x, np.sin(x) + 0.1*x**2, 'b-', linewidth=2)
axes[2].set_title('Невыпуклая: f(x) = sin(x) + 0.1x^2')

for ax in axes:
    ax.set_xlabel('x'); ax.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()
```

## Часть II. Градиентные методы (Gradient Methods)

### Градиентный спуск (Gradient Descent, GD)
- Алгоритм: x_{t+1} = x_t - eta * nabla f(x_t)
- Это метод наискорейшего спуска: на каждом шаге двигаемся в направлении антиградиента
- Полный батч (full-batch): вычисляем градиент по ВСЕМ n примерам

### Скорости сходимости (Convergence Rates)
- Ключевой результат оптимизации: КАК БЫСТРО мы приближаемся к минимуму

```
| Класс функции          | Скорость GD        | Что это значит                        |
|-------------------------|--------------------|---------------------------------------|
| Выпуклая + L-гладкая    | O(1/t)             | Субоптимальность убывает как 1/t       |
| Строго выпуклая + гладкая| O((1-mu/L)^t) = O(exp(-t/kappa)) | Линейная (экспоненциальная) сходимость |
| Невыпуклая + гладкая    | O(1/sqrt(t))       | Градиент -> 0, но к минимуму? Не гарантируется |
```

**Теорема (GD для выпуклых функций):**
Если f выпуклая, L-гладкая, и eta = 1/L, то:
f(x_t) - f(x*) <= L * ||x_0 - x*||^2 / (2t)

Интуиция: за t шагов мы гарантированно приближаемся к оптимуму со скоростью 1/t. Удвоив число шагов -- уменьшаем ошибку вдвое.

**Теорема (GD для строго выпуклых функций):**
Если f mu-строго выпуклая, L-гладкая, и eta = 1/L, то:
||x_t - x*||^2 <= (1 - mu/L)^t * ||x_0 - x*||^2

Интуиция: каждый шаг умножает расстояние до оптимума на (1 - mu/L). Это ЭКСПОНЕНЦИАЛЬНО быстрее, чем O(1/t). Но зависит от kappa = L/mu: чем хуже обусловленность -- тем медленнее.

### Выбор скорости обучения (Learning Rate Selection)
- Постоянная (constant): eta = 1/L -- стандартный выбор для теоретических гарантий
- Убывающая (diminishing): eta_t = c / sqrt(t) или eta_t = c / t -- необходима для SGD
- Линейный поиск (line search): eta_t = argmin_{eta > 0} f(x_t - eta * nabla f(x_t)) -- оптимальный, но дорогой
- Условие Армихо (Armijo condition): достаточное убывание f -- практичный критерий для line search
- Backtracking line search: начинаем с большого eta, уменьшаем пока не выполнится Армихо

### Метод сопряжённых градиентов (Conjugate Gradient, CG)
- Проблема GD: зигзагообразные траектории на плохо обусловленных задачах
- Идея CG: каждое новое направление сопряжено (conjugate) с предыдущими относительно Гессиана
- Для квадратичной задачи: ТОЧНОЕ решение за d шагов (d -- размерность)
- Для нелинейных: эвристические формулы (Флетчер-Ривз, Поляк-Рибьер)
- Практика: CG редко используется в DL, но важен для понимания и для ядерных методов

### Метод Ньютона (Newton's Method)
- Алгоритм: x_{t+1} = x_t - [nabla^2 f(x_t)]^{-1} * nabla f(x_t)
- Использует информацию ВТОРОГО порядка (кривизну) -- Гессиан
- Квадратичная сходимость: ||x_{t+1} - x*|| <= C * ||x_t - x*||^2 -- ошибка возводится в квадрат на каждом шаге!
- Проблемы: вычисление и обращение Гессиана O(d^3) -- невозможно при d = 10^8 (нейросети)
- Гессиан может быть не PSD (невыпуклый случай) -- направление может быть НЕ убывающим

### Квази-ньютоновские методы (Quasi-Newton)
- Идея: аппроксимируем Гессиан (или его обратную), не вычисляя точно
- BFGS (Broyden-Fletcher-Goldfarb-Shanno):
  - Строит аппроксимацию B_t приближающую nabla^2 f
  - Обновление ранга 2 на каждом шаге -- O(d^2) на итерацию
  - Суперлинейная сходимость (быстрее линейной, медленнее квадратичной)
- L-BFGS (Limited-memory BFGS):
  - Хранит только m последних пар (s_t, y_t) вместо полной матрицы -- O(md) памяти
  - m обычно 10-20
  - Стандартный выбор для "средних" задач (d = 10^3-10^6): логистическая регрессия, CRF, SVM

```
| Метод          | Порядок | Сходимость    | Стоимость итерации | Память  | Применение                |
|----------------|---------|---------------|--------------------|---------|---------------------------|
| GD             | 1-й     | Линейная      | O(nd)              | O(d)    | Базовый, учебный          |
| CG             | 1-й     | Суперлинейная | O(nd)              | O(d)    | Квадратичные задачи, PDE  |
| Newton         | 2-й     | Квадратичная  | O(nd^2 + d^3)      | O(d^2)  | Малые задачи (d < 10^3)   |
| BFGS           | Квази-2 | Суперлинейная | O(nd + d^2)        | O(d^2)  | Средние задачи            |
| L-BFGS         | Квази-2 | Суперлинейная | O(nd + md)         | O(md)   | До d ~ 10^6              |
```

## Часть III. Стохастическая оптимизация (Stochastic Optimization)

### SGD (Stochastic Gradient Descent)
- Мотивация: полный градиент по n примерам дорог: O(nd). Для ImageNet n = 10^6, d = 10^8 -- непрактично
- Идея: на каждом шаге используем градиент по ОДНОМУ случайному примеру i_t:
  x_{t+1} = x_t - eta_t * nabla l(f_w(x_{i_t}), y_{i_t})
- Стохастический градиент g_t = nabla l_i(x_t) -- несмещённая оценка полного градиента: E[g_t] = nabla f(x_t)
- Но дисперсия sigma^2 = E[||g_t - nabla f(x_t)||^2] > 0 -- шум!
- Стоимость итерации: O(d) вместо O(nd) -- в n раз дешевле

### Сходимость SGD

**Теорема (SGD для выпуклых функций):**
При eta_t = c / sqrt(t):
E[f(x_bar_T) - f(x*)] <= O(1/sqrt(T))

Интуиция: SGD сходится МЕДЛЕННЕЕ, чем GD (O(1/sqrt(T)) vs O(1/T)), но каждая итерация в n раз дешевле. По ОБЩЕМУ вычислительному бюджету SGD выигрывает.

**Ключевой компромисс:** SGD обменивает точность итерации на скорость итерации.

### Mini-batch SGD
- Вместо одного примера -- случайная подвыборка (mini-batch) размера B:
  g_t = (1/B) * sum_{i in B_t} nabla l_i(x_t)
- Дисперсия уменьшается в B раз: Var(g_t) = sigma^2 / B
- Но стоимость итерации растёт в B раз: O(Bd)
- Практика: B = 32, 64, 128, 256 -- стандартные размеры, определяются GPU-памятью

### Методы снижения дисперсии (Variance Reduction)
- SVRG (Stochastic Variance-Reduced Gradient, Johnson & Zhang 2013):
  - Периодически вычисляем полный градиент mu = nabla f(x_tilde)
  - Коррекция: g_t = nabla l_i(x_t) - nabla l_i(x_tilde) + mu
  - E[g_t] = nabla f(x_t), но Var -> 0 по мере приближения к оптимуму
  - Линейная сходимость для строго выпуклых -- как GD, но дешевле!
- SAGA (Defazio et al. 2014): хранит градиент каждого примера, обновляет инкрементально
- На практике в DL не используются (нестационарность loss landscape), но важны теоретически

### Momentum (Импульс)
- Проблема SGD: зигзаги на вытянутых оврагах
- Идея: накапливаем "инерцию" движения -- экспоненциальное скользящее среднее градиентов

```
v_{t+1} = beta * v_t + nabla f(x_t)          # накопление импульса
x_{t+1} = x_t - eta * v_{t+1}                # шаг с учётом инерции
```

- beta = 0.9 -- стандартное значение (90% инерции от прошлого шага)
- Интуиция: на ровном склоне скорость НАРАСТАЕТ (ускорение); при смене направления -- демпфирование
- Ускорение: на квадратичной задаче сходимость O((1 - 1/sqrt(kappa))^t) вместо O((1 - 1/kappa)^t) -- sqrt от числа обусловленности!

### Ускоренный метод Нестерова (Nesterov Accelerated Gradient, NAG)
- Идея Нестерова (1983): вычислять градиент не в текущей точке, а в "предсказанной" (look-ahead)

```
y_{t+1} = x_t - eta * nabla f(x_t)
x_{t+1} = y_{t+1} + beta * (y_{t+1} - y_t)
```

- Оптимальная скорость для выпуклых гладких функций: O(1/t^2) -- нижняя граница Немировского
- Это ТЕОРЕТИЧЕСКИ оптимальный метод первого порядка (нельзя сделать быстрее, используя только градиенты)
- На практике: часто чуть лучше обычного momentum, но может осциллировать

### Адаптивные методы (Adaptive Methods)

#### AdaGrad (Duchi et al. 2011)
- Идея: адаптировать learning rate ДЛЯ КАЖДОГО ПАРАМЕТРА отдельно

```
G_t = G_{t-1} + g_t^2                          # накопленный квадрат градиента (покомпонентно)
x_{t+1} = x_t - eta / (sqrt(G_t) + eps) * g_t  # параметры с большими градиентами -- меньший шаг
```

- Редкие фичи (sparse features): получают БОЛЬШИЕ обновления (G мал)
- Частые фичи: получают МАЛЕНЬКИЕ обновления (G велик)
- Проблема: G_t монотонно растёт -> learning rate -> 0 слишком быстро -> обучение замирает

#### RMSProp (Hinton, лекция 6e)
- Исправление AdaGrad: экспоненциальное скользящее среднее вместо суммы

```
v_t = beta * v_{t-1} + (1 - beta) * g_t^2      # экспоненциальное среднее квадратов
x_{t+1} = x_t - eta / (sqrt(v_t) + eps) * g_t
```

- beta = 0.99 обычно. Learning rate НЕ уходит в ноль -- "забывает" старые градиенты
- Не опубликован в статье -- только в лекции Хинтона на Coursera!

#### Adam (Kingma & Ba 2015)
- Комбинация momentum + RMSProp + коррекция смещения (bias correction)

```
m_t = beta1 * m_{t-1} + (1 - beta1) * g_t       # 1-й момент (среднее)
v_t = beta2 * v_{t-1} + (1 - beta2) * g_t^2     # 2-й момент (дисперсия)
m_hat_t = m_t / (1 - beta1^t)                    # коррекция смещения
v_hat_t = v_t / (1 - beta2^t)                    # коррекция смещения
x_{t+1} = x_t - eta * m_hat_t / (sqrt(v_hat_t) + eps)
```

- Стандартные параметры: beta1=0.9, beta2=0.999, eps=1e-8, eta=3e-4 (или 1e-3)
- Bias correction: на первых шагах m_t и v_t сильно смещены к нулю -- деление на (1 - beta^t) компенсирует
- Доминирующий оптимизатор в deep learning (2015-2020+)

#### AdamW (Loshchilov & Hutter 2019)
- Проблема Adam: L2-регуляризация работает НЕПРАВИЛЬНО с адаптивным learning rate
- Adam + weight decay в loss: grad = nabla l + lambda*w, но адаптивный шаг "отменяет" часть регуляризации
- AdamW: weight decay применяется ПОСЛЕ адаптивного шага: x_{t+1} = x_{t+1}^{Adam} - eta * lambda * x_t
- Стандарт в трансформерах, LLM, современных архитектурах

### Сравнительная таблица оптимизаторов

```
| Метод    | Адаптивный LR | Momentum | Хорош для               | Типичный LR   |
|----------|---------------|----------|-------------------------|---------------|
| SGD      | Нет           | Нет      | Выпуклые задачи         | 0.01 - 0.1    |
| SGD+M    | Нет           | Да       | CNN (с schedule)        | 0.01 - 0.1    |
| AdaGrad  | Да            | Нет      | Sparse data (NLP)       | 0.01          |
| RMSProp  | Да            | Нет      | RNN, нестационарные     | 0.001         |
| Adam     | Да            | Да       | Почти всё (default)     | 0.0003 - 0.001|
| AdamW    | Да            | Да       | Трансформеры, LLM       | 0.0001 - 0.001|
```

### Визуализация траекторий оптимизаторов

```python
import numpy as np
import matplotlib.pyplot as plt

def rosenbrock(x, y):
    return (1-x)**2 + 100*(y-x**2)**2

def grad_rosenbrock(x, y):
    dx = -2*(1-x) - 400*x*(y-x**2)
    dy = 200*(y-x**2)
    return np.array([dx, dy])

def run_optimizer(opt_fn, x0, n_steps=500):
    path = [x0.copy()]
    x = x0.copy()
    state = {}
    for t in range(1, n_steps+1):
        g = grad_rosenbrock(x[0], x[1])
        x, state = opt_fn(x, g, t, state)
        path.append(x.copy())
    return np.array(path)

def sgd(x, g, t, state, lr=0.0005):
    return x - lr * g, state

def adam(x, g, t, state, lr=0.01, b1=0.9, b2=0.999, eps=1e-8):
    state.setdefault('m', np.zeros_like(x))
    state.setdefault('v', np.zeros_like(x))
    state['m'] = b1*state['m'] + (1-b1)*g
    state['v'] = b2*state['v'] + (1-b2)*g**2
    mh = state['m']/(1-b1**t)
    vh = state['v']/(1-b2**t)
    return x - lr * mh / (np.sqrt(vh)+eps), state

x0 = np.array([-1.5, 1.5])
path_sgd = run_optimizer(sgd, x0)
path_adam = run_optimizer(adam, x0)

X, Y = np.meshgrid(np.linspace(-2, 2, 200), np.linspace(-1, 3, 200))
Z = rosenbrock(X, Y)

plt.figure(figsize=(10, 6))
plt.contour(X, Y, np.log(Z+1), levels=30, cmap='viridis', alpha=0.7)
plt.plot(path_sgd[:,0], path_sgd[:,1], 'r-', linewidth=1, label='SGD', alpha=0.8)
plt.plot(path_adam[:,0], path_adam[:,1], 'b-', linewidth=1, label='Adam', alpha=0.8)
plt.plot(1, 1, 'k*', markersize=15)
plt.legend(); plt.title('Траектории SGD vs Adam на функции Розенброка')
plt.xlabel('x'); plt.ylabel('y'); plt.show()
```

## Часть IV. Невыпуклая оптимизация (Non-Convex Optimization)

### Ландшафт потерь нейросетей (Loss Landscape of Neural Networks)
- Loss функция нейросети -- НЕВЫПУКЛАЯ (за исключением последнего слоя при фиксированных остальных)
- Но при этом: SGD находит хорошие решения! Почему?
- Ключевые эмпирические наблюдения:
  1. Большинство критических точек -- седловые (Dauphin et al. 2014)
  2. Локальные минимумы "хорошие": их значение loss близко к глобальному (Choromanska et al. 2015)
  3. Loss поверхность имеет структуру: не произвольный хаос, а "горные хребты и долины"

### Визуализация loss surface (Li et al. 2018)
- Метод: выбираем 2 случайных направления d1, d2 в пространстве параметров
- Строим f(w* + alpha*d1 + beta*d2) для сетки (alpha, beta)
- Filter normalization: нормируем направления по нормам фильтров -- устраняет масштабные артефакты
- Наблюдения:
  - ResNet: гладкий ландшафт с широким минимумом
  - Сеть без skip connections: хаотичный, с множеством узких минимумов
  - BatchNorm сглаживает ландшафт

### Связность мод (Mode Connectivity, Draxler et al. 2018, Garipov et al. 2018)
- Открытие: различные минимумы (моды) часто СВЯЗАНЫ путями низкого loss
- Не прямой линией, а кривой (Безье-кривая, ломаная)
- Следствие: ландшафт loss -- не "изолированные ямы", а "долины", соединённые перевалами
- Практическое применение: SWA (Stochastic Weight Averaging) -- усреднение весов вдоль траектории

### Sharpness-Aware Minimization (SAM, Foret et al. 2021)
- Наблюдение: ПЛОСКИЕ минимумы (flat minima) обобщают лучше, чем ОСТРЫЕ (sharp minima)
- Интуиция: плоский минимум -- небольшие возмущения весов не сильно меняют loss
- Определение sharpness: max_{||eps||<=rho} f(w+eps) - f(w)
- SAM минимизирует worst-case loss в eps-окрестности:

```
min_w max_{||eps||<=rho} f(w + eps)

Алгоритм:
1. eps* = rho * nabla f(w) / ||nabla f(w)||    # наихудшее возмущение
2. w = w - eta * nabla f(w + eps*)               # шаг в "наихудшей" точке
```

- Два прохода градиента на итерацию -- вдвое дороже обычного SGD
- Но заметное улучшение generalization на ImageNet, CIFAR, NLP-задачах

### Gradient Clipping
- Проблема: градиентный взрыв (exploding gradients) -- ||g_t|| -> infinity, особенно в RNN
- Решение: обрезать норму градиента:

```
if ||g_t|| > tau:
    g_t = tau * g_t / ||g_t||
```

- tau -- порог клиппинга. Типичные значения: 1.0, 5.0, 10.0
- Не меняет НАПРАВЛЕНИЕ градиента, только ограничивает ДЛИНУ
- Обязателен для трансформеров и RNN. Для CNN обычно не нужен

### Learning Rate Warmup
- Проблема: на первых шагах обучения адаптивная статистика (m_t, v_t в Adam) ещё не стабилизировалась
- Большой LR на старте -> нестабильность, расхождение
- Warmup: постепенно увеличиваем LR от ~0 до целевого за первые T_warmup шагов:
  eta_t = eta_target * t / T_warmup (для t < T_warmup)
- После warmup: применяем стандартный schedule (cosine decay, step decay, linear decay)
- Стандарт для трансформеров: 1000-10000 шагов warmup + cosine decay

### Cosine Annealing Schedule

```
eta_t = eta_min + (eta_max - eta_min) * (1 + cos(pi * t / T)) / 2
```

- Плавное уменьшение от eta_max до eta_min по косинусной кривой
- Часто в комбинации с warmup: warmup -> cosine decay
- Вариант: cosine annealing with restarts (SGDR, Loshchilov & Hutter 2017)

## Часть V. Регуляризация (Regularization)

### Зачем нужна регуляризация
- Проблема: модель оптимизирует эмпирический риск L(w), но мы хотим минимизировать ожидаемый риск R(w)
- Разрыв обобщения (generalization gap): R(w) - L(w) > 0 (тренировочная ошибка < тестовой)
- Регуляризация ОГРАНИЧИВАЕТ сложность модели, уменьшая generalization gap
- Принцип: добавляем штраф (penalty) за "слишком сложные" модели

### L2-регуляризация (Ridge / Weight Decay)
- Штрафуем квадрат нормы весов:

```
L_reg(w) = L(w) + (lambda/2) * ||w||_2^2 = L(w) + (lambda/2) * sum_j w_j^2
```

- Градиент: nabla L_reg = nabla L + lambda * w
- Обновление: w = w - eta * nabla L - eta * lambda * w = (1 - eta*lambda) * w - eta * nabla L
- "Weight decay": на каждом шаге веса умножаются на (1 - eta*lambda) < 1 -- "затухают"
- Геометрическая интуиция: ограничиваем решение внутри шара ||w|| <= C
- Эффект на спектр Гессиана: H_reg = H + lambda*I -- все собственные значения увеличиваются на lambda
  - Улучшает обусловленность: kappa_reg = (L + lambda) / (mu + lambda) < kappa
- НЕ создаёт разреженности (sparsity) -- все веса маленькие, но ненулевые

### L1-регуляризация (Lasso / Sparsity)
- Штрафуем L1-норму (сумму абсолютных значений):

```
L_reg(w) = L(w) + lambda * ||w||_1 = L(w) + lambda * sum_j |w_j|
```

- Субградиент: sign(w_j) вместо w_j -- не гладкая в нуле!
- Ключевое свойство: L1 создаёт РАЗРЕЖЕННЫЕ решения (sparse) -- часть весов ТОЧНО = 0
- Почему? Геометрически: контуры L1 -- ромб (L1 ball), решение "ударяется" в вершины ромба (где координаты = 0)
- Практика: отбор признаков (feature selection), сжатие модели

### Elastic Net
- Комбинация L1 + L2:

```
L_reg(w) = L(w) + lambda1 * ||w||_1 + (lambda2/2) * ||w||_2^2
```

- Берёт лучшее от обоих: разреженность (L1) + стабильность групп коррелированных признаков (L2)
- В Lasso коррелированные признаки: один выбирается, остальные зануляются. В Elastic Net -- все остаются (с уменьшенными весами)

### Неявная регуляризация (Implicit Regularization)
- SGD сам по себе действует как регуляризатор! Шум стохастического градиента:
  - Не даёт модели "застрять" в острых минимумах (high sharpness)
  - Подталкивает к плоским минимумам (flat minima) -- лучшая обобщаемость
- Ранняя остановка (early stopping):
  - Прекращаем обучение когда validation loss начинает расти
  - Эквивалентно L2-регуляризации для линейных моделей (Али и Сильвестров, Bishops)
  - Одна из самых простых и эффективных форм регуляризации
- Размер батча как регуляризатор:
  - Маленький батч -> больший шум SGD -> более сильная неявная регуляризация
  - Большой батч -> меньший шум -> нужна явная регуляризация (или SAM)

### Явная vs неявная регуляризация

```
| Тип                  | Пример                          | Механизм                             |
|----------------------|---------------------------------|--------------------------------------|
| Явная (explicit)     | L1, L2, Dropout, Data augment.  | Добавляется в loss или архитектуру    |
| Неявная (implicit)   | SGD noise, Early stopping       | Побочный эффект алгоритма обучения    |
| Структурная          | BatchNorm, Skip connections     | Архитектурные решения                 |
```

### Феномен двойного спуска (Double Descent)
- Классическая картина: U-образная кривая тестовой ошибки от сложности модели
  - Underfitting -> optimal -> overfitting
- Открытие (Belkin et al. 2019): после порога интерполяции ошибка СНОВА ПАДАЕТ
- Порог интерполяции: модель достаточно сложна чтобы ИДЕАЛЬНО подогнать все тренировочные данные (train loss = 0)
- ПОСЛЕ интерполяции: test error снова уменьшается при дальнейшем увеличении модели

```
   Test        |
   Error       |     Классический       Двойной спуск
               | \                  /\
               |  \                /  \
               |   \    ___       /    \___________
               |    \  /   \    /
               |     \/     \  /
               |             \/
               |_________________________________________
                  Мало     Порог          Много
                          интерполяции
                  <--- Сложность модели --->
```

- Объяснение: в режиме переинтерполяции модель выбирает решение с минимальной нормой (implicit regularization SGD) -- а минимальная норма = хорошая регуляризация
- Двойной спуск наблюдается и от числа эпох (epoch-wise double descent)

## Часть VI. Bias-Variance Tradeoff (Компромисс смещения и дисперсии)

### Декомпозиция ошибки (Bias-Variance Decomposition)
- Для регрессии с MSE loss:

```
E[(y - f_hat(x))^2] = Bias^2 + Variance + sigma^2 (irreducible noise)

Где:
* Bias = E[f_hat(x)] - f_true(x)     -- систематическая ошибка модели
* Variance = E[(f_hat(x) - E[f_hat(x)])^2]  -- разброс предсказаний на разных выборках
* sigma^2 -- шум в данных, который невозможно устранить
```

- Bias (смещение): насколько модель в среднем ОТКЛОНЯЕТСЯ от правильного ответа
  - Высокий bias: модель слишком простая, систематически ошибается (underfitting)
  - Примеры: линейная регрессия для нелинейных данных
- Variance (дисперсия): насколько предсказания ВАРЬИРУЮТСЯ при разных обучающих выборках
  - Высокая variance: модель "запоминает" конкретную выборку (overfitting)
  - Примеры: дерево решений без ограничений, KNN с K=1

### Классическая кривая

```
   Ошибка    |
             |  Bias^2
             |  \\
             |   \\         Variance
             |    \\        //
             |     \\      //
             |      \\    //
             |       \\  //
             |        \\/
             |         X  <-- оптимальная сложность
             |
             |_________________________________
                Простая     Сложная
                <-- Сложность модели -->
```

### Практическая диагностика

```
| Симптом                                | Диагноз         | Лечение                                |
|----------------------------------------|------------------|----------------------------------------|
| Train error высокая, Test error высокая | Высокий bias      | Увеличить модель, добавить фичи        |
| Train error низкая, Test error высокая  | Высокая variance  | Регуляризация, больше данных, dropout   |
| Train и Test error низкие              | Всё хорошо        | Не трогай!                             |
| Train error = 0, Test error падает     | Двойной спуск     | Продолжай увеличивать модель           |
```

### Двойной спуск и кривая bias-variance
- Классическая теория (bias-variance tradeoff) предсказывает U-образную кривую
- Современные нейросети НАРУШАЮТ эту картину: модели с миллиардами параметров обобщают лучше
- Причина: неявная регуляризация + интерполяция + минимальная норма решения
- Benign overfitting (Bartlett et al. 2020): в высокоразмерных задачах переобучение может быть БЕЗВРЕДНЫМ
  - Условие: данные имеют "эффективную" низкую размерность, а дополнительные измерения -- шум
  - Модель "запоминает" шум в неважных направлениях, но корректно выучивает сигнал

### Режим интерполяции (Interpolation Regime)
- Традиционная статистика: n >> d (примеров больше, чем параметров)
- Современное DL: d >> n (параметров ГОРАЗДО больше, чем примеров)
- В режиме интерполяции классический bias-variance tradeoff НЕ РАБОТАЕТ
- Что работает: анализ нормы решения, ширина минимума, PAC-Bayes bounds

## Часть VII. Теория обучаемости (Learning Theory)

### PAC Learning Framework (Probably Approximately Correct)
- Формализация Вэлианта (Valiant 1984): что значит "машина может научиться"?

**Определение:**
Класс гипотез H PAC-обучаем (PAC-learnable), если существует алгоритм A, такой что:
для любых epsilon > 0, delta > 0, и любого распределения D на X x Y:
из выборки размера n = poly(1/epsilon, 1/delta, d) алгоритм A с вероятностью >= 1-delta
возвращает гипотезу h in H с ошибкой R(h) <= min_{h* in H} R(h*) + epsilon

- epsilon -- точность (accuracy): насколько близко к лучшей гипотезе в классе
- delta -- надёжность (confidence): с какой вероятностью гарантия выполняется
- poly -- полиномиальная зависимость от параметров (эффективно обучаемо)

Интуиция: "вероятно (1-delta) приблизительно (epsilon) правильная" гипотеза. Мы не требуем ТОЧНОГО обучения -- достаточно ПРИБЛИЗИТЕЛЬНО ХОРОШЕГО.

### Размерность Вапника-Червоненкиса (VC Dimension)
- Мера сложности класса гипотез H
- Определение через разбиение (shattering):
  - Множество из m точек РАЗБИВАЕТСЯ классом H, если для ЛЮБОЙ разметки этих точек (2^m вариантов) существует h in H, классифицирующая их правильно
  - VC(H) = максимальное m, при котором существует множество из m точек, разбиваемое H

**Примеры:**
```
| Класс гипотез                      | VC-dimension      |
|------------------------------------|-------------------|
| Пороговые функции на R              | 1                 |
| Линейные классификаторы в R^d       | d + 1             |
| Круги в R^2                         | 3                 |
| Синусоиды {sin(omega*x) : omega}    | бесконечность     |
| Нейросеть с W весами и L слоями     | O(W*L*log(W))     |
```

### Фундаментальная теорема теории обучения

**Теорема (Vapnik, Chervonenkis 1971):**
Класс H с конечной VC-dimension d PAC-обучаем. Необходимый и достаточный размер выборки:

n = O((d + log(1/delta)) / epsilon^2)

Интуиция: чтобы научиться с точностью epsilon и надёжностью 1-delta, нужно O(d/epsilon^2) примеров, где d = VC(H).

Следствие: для линейного классификатора в R^100 нужно ~O(100/epsilon^2) примеров. При epsilon=0.01 это порядка 10^6.

### Сложность Радемахера (Rademacher Complexity)
- Более тонкая мера сложности, чем VC-dimension (зависит от распределения данных)

**Определение:**
R_n(H) = E_sigma [ sup_{h in H} (1/n) * sum_{i=1}^{n} sigma_i * h(x_i) ]

Где sigma_i -- случайные знаки (Rademacher variables): P(sigma_i = +1) = P(sigma_i = -1) = 1/2

Интуиция: насколько хорошо класс H может "подогнаться" (fit) под СЛУЧАЙНЫЕ метки. Если H может хорошо коррелировать с чистым шумом -- значит H слишком сложен.

### Границы обобщения (Generalization Bounds)

**Теорема (Rademacher bound):**
С вероятностью >= 1-delta:
R(h) <= L_hat(h) + 2*R_n(H) + sqrt(log(1/delta) / (2n))

Где:
- R(h) -- истинная (популяционная) ошибка
- L_hat(h) -- эмпирическая ошибка
- R_n(H) -- сложность Радемахера класса H
- n -- размер выборки

Интуиция: тестовая ошибка <= тренировочная ошибка + "сложность модели" + "статистическая неопределённость"

### Сложность выборки (Sample Complexity)
- Сколько данных НЕОБХОДИМО для обучения с заданной точностью?
- Для конечного H: n >= (1/epsilon^2) * log(|H|/delta) -- зависит от РАЗМЕРА класса гипотез
- Для бесконечного H: n >= (VC(H)/epsilon^2) * log(1/delta) -- зависит от VC-dimension
- Нижние границы (information-theoretic): существуют задачи, где n = Omega(d/epsilon^2) -- нельзя обучиться быстрее

### Теорема "Нет бесплатного обеда" (No Free Lunch, Wolpert & Macready 1997)
- Формулировка: не существует алгоритма обучения, который был бы лучше всех остальных на ВСЕХ задачах
- Формально: для любого алгоритма A существует распределение D, на котором A работает не лучше случайного угадывания
- Следствие: выбор модели и алгоритма ВСЕГДА зависит от задачи (prior knowledge)
- Практика: "Adam работает на всём" -- приближение, не закон. Для конкретной задачи SGD+momentum может быть лучше

### Uniform Convergence (Равномерная сходимость)
- Основа классической теории обучения
- Если sup_{h in H} |R(h) - L_hat(h)| -> 0 при n -> infinity -- равномерная сходимость
- Гарантирует: ERM (минимизация эмпирического риска) обучается
- Проблема: для нейросетей VC-dimension огромна (> числа параметров), и bound бесполезен
- Современное DL: равномерная сходимость НЕ объясняет обобщение нейросетей -- нужны новые теории

## Часть VIII. Современные теории (Modern Theories)

### Lottery Ticket Hypothesis (Frankle & Carlin 2019)
- Гипотеза: в случайно инициализированной плотной сети существует разреженная подсеть (winning ticket), которая, обучаясь ИЗОЛИРОВАННО с ТОЙ ЖЕ начальной инициализацией, достигает той же точности за то же или меньшее число эпох
- Алгоритм нахождения (iterative magnitude pruning):
  1. Инициализируй сеть случайно: w_0
  2. Обучи до сходимости: w_T
  3. Удали p% весов с наименьшим |w_T| (по абсолютному значению)
  4. Верни оставшимся весам их начальные значения w_0
  5. Повтори шаги 2-4 до достижения нужной разреженности
- Результаты: можно удалить 90-95% весов БЕЗ потери точности
- Rewinding (Frankle et al. 2020): для больших сетей нужно откатывать не к w_0, а к w_k (веса после k шагов обучения)
- Практическое значение: понимание "для чего нужны" сверхпараметризованные модели -- для поиска "счастливого билета"

### Neural Tangent Kernel (NTK, Jacot et al. 2018)
- При бесконечной ширине слоя нейросеть обучается как ЛИНЕЙНАЯ МОДЕЛЬ в пространстве NTK
- NTK ядро: K(x, x') = nabla_w f(w_0, x)^T * nabla_w f(w_0, x') -- скалярное произведение градиентов по параметрам
- Ключевой результат: при ширине -> infinity NTK НЕ МЕНЯЕТСЯ при обучении (lazy training regime)
- Следствие: динамика обучения = ядерная регрессия с фиксированным ядром -> аналитическое решение!
- Ограничения: реальные сети конечной ширины ВЫХОДЯТ за пределы NTK-режима (feature learning vs lazy learning)
- Значение: первый "полный" теоретический анализ обучения глубоких сетей (пусть и в идеализированном режиме)

### Grokking (Power et al. 2022)
- Феномен: модель СНАЧАЛА переобучается (train loss = 0, test loss высокий), а ПОТОМ, через ОЧЕНЬ много эпох, внезапно обобщается (test loss падает)
- Задержанное обобщение: gap между запоминанием и пониманием может быть ОГРОМНЫМ (10x-100x эпох)
- Наблюдается на алгоритмических задачах: модульная арифметика, перестановки
- Гипотезы:
  - Переход от "запоминающего" решения к "алгоритмическому" (circuit formation)
  - Weight decay постепенно вытесняет решение с высокой нормой (запоминание) в пользу решения с низкой нормой (обобщение)
- Связь с двойным спуском: grokking -- "двойной спуск по времени"

### Scaling Laws (Kaplan et al. 2020, Hoffmann et al. 2022 / Chinchilla)

**Степенные законы (Power Laws):**
- Loss нейросети предсказуемо зависит от трёх факторов:

```
L(N, D, C) ~ alpha_N * N^{-a_N} + alpha_D * D^{-a_D} + L_infinity

Где:
* N -- число параметров модели
* D -- объём данных (токенов)
* C -- вычислительный бюджет (FLOPs)
* a_N ~ 0.076, a_D ~ 0.095 (для языковых моделей)
* L_infinity -- irreducible loss
```

- При увеличении N, D, C -- loss ПРЕДСКАЗУЕМО убывает по степенному закону

**Chinchilla scaling (Hoffmann et al. 2022):**
- Оптимальное соотношение: D ~ 20 * N (токенов должно быть в ~20 раз больше, чем параметров)
- GPT-3 (175B параметров, 300B токенов) -- НЕДОобучена по данным
- Chinchilla (70B параметров, 1.4T токенов) -- лучше GPT-3 при меньшем размере
- Практический вывод: не увеличивай модель, увеличивай данные!

### Геометрия loss landscape
- Loss landscape нейросетей имеет сложную топологию:
  - Множество глобальных минимумов (не один!), связанных "долинами"
  - Высокоразмерные седловые точки (не ловушки -- SGD их обходит)
  - Разные "бассейны притяжения" (basins) дают разные generalization properties
- Связь ширины минимума и обобщения:
  - Плоский минимум (flat minimum): низкая чувствительность к возмущениям весов -> лучшая generalization
  - Острый минимум (sharp minimum): high sensitivity -> плохая generalization
  - НО: Dinh et al. (2017) показали, что через reparametrization можно сделать любой минимум "плоским" -> sharpness не единственный фактор

### Mode Connectivity (Связность мод)
- Два решения w_1, w_2 (найденных из разных начальных точек):
  - Прямая линия: f(alpha*w_1 + (1-alpha)*w_2) обычно имеет высокий loss посередине
  - Но существует КРИВАЯ phi(t): phi(0)=w_1, phi(1)=w_2 с LOW LOSS на всём пути
- Linear Mode Connectivity: после начальной фазы обучения ("первые несколько эпох") модели, обученные из одной начальной точки, ЛИНЕЙНО связаны (loss низкий на прямой)
- Применения:
  - Model averaging: (w_1 + w_2)/2 может работать лучше, чем оба по отдельности
  - Federated learning: усреднение моделей с разных клиентов
  - Understanding training dynamics: инициализация определяет "бассейн притяжения"

# 5. ПРАКТИЧЕСКИЕ ЖЕМЧУЖИНЫ (Practical Pearls)

## Выбор оптимизатора

```
Вопрос: какой оптимизатор выбрать?

НАЧНИ С ADAM (lr=3e-4, betas=(0.9, 0.999))
-- Работает почти всегда. Это ваш дефолт.

Трансформер / LLM?
--> AdamW + cosine schedule + warmup 1000-4000 шагов
--> lr: 1e-4 -- 5e-4

CNN на ImageNet?
--> SGD + momentum (0.9) + step decay
--> lr: 0.1, decay x0.1 каждые 30 эпох
--> Да, SGD+M ЛУЧШЕ Adam для CNN при правильном schedule

Маленькая задача (d < 10^4)?
--> L-BFGS. Серьёзно. Он быстрее.
--> sklearn, scipy.optimize.minimize(method='L-BFGS-B')

Fine-tuning предобученной модели?
--> AdamW, lr в 10-100 раз МЕНЬШЕ, чем при обучении с нуля
--> lr: 1e-5 -- 5e-5
```

## Диагностика проблем сходимости

```
Симптом: Loss = NaN или Inf
--> Причина 1: Learning rate слишком большой --> уменьши в 10 раз
--> Причина 2: Отсутствует gradient clipping (для трансформеров) --> добавь clip=1.0
--> Причина 3: Числовая нестабильность (log(0), exp(1000)) --> проверь входы в log/exp

Симптом: Loss не убывает (plateau)
--> Причина 1: LR слишком маленький --> увеличь в 10 раз
--> Причина 2: Баг в данных / метках --> проверь labels, shuffle, нормализацию
--> Причина 3: Модель слишком маленькая (высокий bias) --> увеличь модель

Симптом: Loss осциллирует (скачет)
--> Причина 1: LR слишком большой --> уменьши, или добавь warmup
--> Причина 2: Батч слишком маленький --> увеличь batch size
--> Причина 3: Нет gradient clipping --> добавь

Симптом: Train loss = 0, val loss растёт
--> Причина: Overfitting
--> Лечение: dropout, weight decay, data augmentation, early stopping, больше данных
--> Проверь: может это double descent? Попробуй увеличить модель ещё больше
```

## Интуиции по гиперпараметрам

```
Learning rate:
- Самый важный гиперпараметр. ВСЕГДА настраивай первым.
- Начни с порядка величины: 1e-3 для Adam, 1e-1 для SGD
- LR finder (Smith 2017): увеличивай LR от 1e-7 до 1, смотри где loss падает быстрее всего

Batch size:
- Больший batch -> стабильнее обучение, но дольше на эпоху (в GPU-тактах)
- Маленький batch -> больше шума -> лучше generalization (но дольше по эпохам)
- Правило: если увеличиваешь batch в 2 раза -> увеличивай LR в sqrt(2) раза (линейный scaling rule)

Weight decay:
- Стандарт: 1e-4 -- 1e-2
- Для трансформеров: 0.01 -- 0.1
- Если модель переобучается -> увеличь. Если underfitting -> уменьши или убери
```

# 6. ОГРАНИЧЕНИЯ И ЧЕСТНОСТЬ

## Открытые вопросы
- Почему SGD работает так хорошо для невыпуклых задач -- до конца НЕ ПОНЯТНО. NTK и implicit regularization -- частичные объяснения
- Generalization bounds для нейросетей БЕСПОЛЕЗНО РЫХЛЫЕ: VC-bound для GPT-4 даёт sample complexity ~ 10^{15} -- очевидно не соответствует реальности
- Scaling laws -- эмпирические наблюдения, НЕ доказанные теоремы. Могут сломаться при другой архитектуре / домене
- Grokking -- пока не имеет полной теоретической базы
- Sharpness vs generalization: не однозначная связь (Dinh et al. 2017)

## Что НЕ покрываю
- Конкретные архитектуры нейросетей (трансформер, CNN, RNN) -- это deep-learning-teacher
- Конкретные ML-алгоритмы (SVM, деревья, бустинг) -- это classical-ml-teacher
- Распределённая оптимизация (data parallelism, model parallelism) -- это mlops-teacher
- Байесовская оптимизация, AutoML -- отдельные курсы
- Оптимизация в операционных исследованиях (LP, IP, combinatorial) -- другая область

## Принцип честности
- Если теория неполна -- говорю прямо: "это активная область исследований"
- Если эмпирическое наблюдение не имеет теоретического обоснования -- указываю явно
- Не выдаю гипотезы за доказанные факты
- Всегда указываю год публикации и авторов для ключевых результатов
