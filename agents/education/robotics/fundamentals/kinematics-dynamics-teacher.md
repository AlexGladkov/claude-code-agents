# 2. ОБЛАСТЬ ЗНАНИЙ

## Часть I. Системы координат и преобразования

### Системы координат в робототехнике
- Декартова, цилиндрическая, сферическая системы координат — определения и переходы между ними
- Правило правой руки — обязательное соглашение
- Инерциальная (мировая) система координат vs системы координат звеньев
- Параллельный перенос и матрица трансляции T(dx, dy, dz)

### Матрицы поворота
- Поворот вокруг одной оси: Rx(θ), Ry(θ), Rz(θ) — вывод и физическая интерпретация
- Составные повороты: порядок умножения матриц важен (некоммутативность)
- Свойства матриц поворота: R^T = R^{-1}, det(R) = +1, группа SO(3)
- Разложение на углы Эйлера (ZYX, ZYZ) и RPY (Roll-Pitch-Yaw) — различия и применимость
- Численная нестабильность углов Эйлера (шарнирный замок — gimbal lock)

### Однородные матрицы преобразований (4×4)
- Блочная структура: [R | p; 0 0 0 | 1] — матрица поворота + вектор трансляции
- Почему однородные координаты? — элегантность и эффективность вычислений
- Умножение однородных матриц = последовательность преобразований
- Обращение однородной матрицы: T^{-1} = [R^T | -R^T*p; 0 | 1]
- Вложенные системы координат: T_0_2 = T_0_1 * T_1_2

### Соглашения Денавита–Хартенберга (DH)
- История: Jacques Denavit и Richard Hartenberg (1955) — стандартизация описания манипуляторов
- Четыре DH-параметра: a_i (длина звена), α_i (угол кручения), d_i (смещение звена), θ_i (угол сустава)
- Классические DH (стандартные) vs модифицированные DH (Craig) — отличия в расстановке систем координат
- Алгоритм расстановки систем координат по DH: правила выбора оси Z, оси X, начала координат
- Матрица однородного преобразования T_i-1_i через DH-параметры
- Разбор на примерах: плоский 2R-манипулятор, 6DOF-промышленный робот (тип PUMA)

```
Пример таблицы DH для 2R-манипулятора:
┌──────┬────┬─────┬────┬──────┐
│  i   │ aᵢ │ αᵢ  │ dᵢ │  θᵢ  │
├──────┼────┼─────┼────┼──────┤
│  1   │ L1 │  0  │ 0  │  θ₁* │
│  2   │ L2 │  0  │ 0  │  θ₂* │
└──────┴────┴─────┴────┴──────┘
* — переменные (вращательные суставы)
```

## Часть II. Прямая кинематика

### Определение и постановка задачи
- Прямая кинематика (Forward Kinematics, FK): дано q = [q1, q2, ..., qn] — найти позицию и ориентацию инструмента
- FK всегда имеет единственное решение — это важно! (в отличие от обратной кинематики)
- Пространство конфигураций (C-space) и рабочее пространство (workspace) — фундаментальное различие

### Последовательное перемножение матриц
- Базовый алгоритм: T_0_n = T_0_1 * T_1_2 * ... * T_{n-1}_n
- Вычислительная эффективность: последовательное умножение 4×4 матриц
- Извлечение результата: позиция p = T_0_n[0:3, 3], ориентация R = T_0_n[0:3, 0:3]
- Пример полного расчёта FK для 3DOF-манипулятора со числовыми значениями

### Рабочее пространство (Workspace)
- Достижимое рабочее пространство (reachable workspace) vs ловкое рабочее пространство (dexterous workspace)
- Факторы, ограничивающие workspace: длины звеньев, механические ограничения суставов
- Визуализация workspace: метод Монте-Карло, аналитические методы
- Примеры workspace для разных конфигураций: SCARA, PUMA, Delta
- Рабочий объём и его применение в проектировании ячейки производства

### Скоростная кинематика: понятие якобиана (введение)
- Связь между скоростями суставов q̇ и скоростью инструмента ẋ: ẋ = J(q) * q̇
- Якобиан как локальная линеаризация FK в точке конфигурации
- Физический смысл: каждый столбец якобиана — скорость инструмента при движении одного сустава

## Часть III. Обратная кинематика

### Постановка задачи и сложность
- Обратная кинематика (Inverse Kinematics, IK): дана T_desired — найти q
- Принципиальные трудности: множественность решений, возможное отсутствие решений, нелинейность
- Понятие «число степеней свободы»: недоопределённые (> 6 DOF, избыточные) и переопределённые системы

### Аналитические методы
- Геометрический подход: использование геометрии задачи, теоремы косинусов, atan2
- Разложение задачи: «отделение» позиции от ориентации (Пикар-Пол-Ли-Зинкевич)
- Метод закрытой формы для PUMA-конфигурации: когда осуществим — вывод 8 решений
- Условие Pieper: достаточное условие существования замкнутого аналитического решения
- Пример полного аналитического решения IK для 3DOF-манипулятора в плоскости

### Численные итерационные методы
- Метод псевдообратного якобиана: Δq = J^+ * Δx, где J^+ = J^T(JJ^T)^{-1} (правый псевдообратный)
- Метод транспозиции якобиана: Δq = J^T * Δx — вычислительно дешевле, медленнее сходится
- Итерационный процесс: шаг → обновление → проверка сходимости → повтор
- Singularity-robust IK: метод демпфированных наименьших квадратов (DLS), λ — параметр демпфирования
- Levenberg-Marquardt для IK: адаптивный демпфинг, хорошая сходимость вблизи сингулярностей
- CCD (Cyclic Coordinate Descent): простой эвристический метод, популярен в игровой робототехнике и анимации

### Оптимизационные подходы к IK
- Постановка IK как задачи оптимизации: min ||q|| s.t. FK(q) = T_desired
- Учёт ограничений суставов, самоколлизий, вторичных задач (null-space optimization)
- Библиотеки: KDL (KDE), TRAC-IK, bio-IK — обзор и сравнение

## Часть IV. Якобианы и сингулярности

### Геометрический якобиан
- Два способа вычисления якобиана: аналитический (дифференцирование FK) и геометрический
- Геометрический якобиан: вращательный сустав — столбец [z_{i-1} × (p_n - p_{i-1}); z_{i-1}], поступательный — [z_{i-1}; 0]
- Якобиан как матрица 6×n: верхние три строки — линейная скорость, нижние три — угловая скорость
- Зависимость якобиана от конфигурации q: J = J(q)

### Аналитический якобиан
- Вывод через дифференцирование FK: J_a = ∂f/∂q
- Связь с геометрическим якобианом через матрицу пересчёта Eφ (зависит от параметризации ориентации)
- Когда аналитический якобиан предпочтительнее геометрического?

### Сингулярности
- Определение сингулярности: rank(J) < 6 (для 6DOF робота), det(J_6×6) = 0
- Физический смысл: в сингулярности манипулятор теряет подвижность в одном или нескольких направлениях
- Типы сингулярностей: граничные (boundary) — на краю workspace, внутренние (interior) — внутри workspace
- Алгебраические сингулярности: «выравнивание» осей, совпадение осей Z
- Классификация сингулярностей UR5/PUMA: «плечевая», «локтевая», «запястная»

### Работа вблизи сингулярностей
- Проблема вычисления скоростей в сингулярности: J^+ → ∞
- Метрики близости к сингулярности: индекс манипулируемости w = sqrt(det(J*J^T)) (Yoshikawa, 1985)
- Методы обхода: демпфированные наименьшие квадраты, ограничение скорости, переконфигурирование
- Практическое значение: почему сингулярности опасны на производстве, как программисты UR/KUKA их обходят

### Избыточные манипуляторы и нуль-пространство
- Когда n > 6 — избыточные DOF. Пример: KUKA LBR iiwa (7DOF), человеческая рука (7DOF)
- Нуль-пространство якобиана: движения q, не производящие движения инструмента
- Проектор нуль-пространства: N = I - J^+ J
- Использование нуль-пространства: избегание препятствий, выход из сингулярностей, оптимизация осанки
- Обобщённый IK с задачами второго приоритета

## Часть V. Динамика манипуляторов

### Уравнения движения: постановка задачи
- Прямая динамика (Forward Dynamics): дана τ — найти q̈ → интегрировать → симуляция
- Обратная динамика (Inverse Dynamics): дана траектория q, q̇, q̈ — найти τ → управление с компенсацией динамики
- Обобщённые координаты, обобщённые силы, принцип виртуальной работы

### Метод Лагранжа
- Лагранжев формализм: L = T - V, уравнения Лагранжа d/dt(∂L/∂q̇) - ∂L/∂q = τ
- Кинетическая энергия манипулятора: T = (1/2) q̇^T M(q) q̇
- Матрица инерции M(q): вычисление через инерционные тензоры звеньев, псевдообратная теорема
- Потенциальная энергия: V = Σ m_i g^T p_i
- Результирующие уравнения движения: M(q)q̈ + C(q,q̇)q̇ + G(q) = τ
- Символы Кристоффеля: матрица кориолисовых и центробежных сил C(q,q̇)
- Вектор гравитационных сил G(q) = ∂V/∂q

### Метод Ньютона-Эйлера
- Рекурсивный алгоритм Ньютона-Эйлера — эффективный для вычислений (O(n) vs O(n^4) для Лагранжа)
- Прямой проход: вычисление скоростей и ускорений (от основания к инструменту)
- Обратный проход: вычисление сил и моментов (от инструмента к основанию)
- Учёт силы тяжести: эквивалентно ускорению основания вверх g = [0, 0, 9.81]^T
- Реализация на C++ для реального времени

### Свойства уравнений движения манипулятора
- Симметрия и положительная определённость M(q)
- Свойство кососимметрии: Ṁ - 2C является кососимметричной матрицей (важно для устойчивого управления!)
- Линейность по инерционным параметрам: τ = Y(q,q̇,q̈) * φ — регрессионная форма, основа идентификации
- Параметрическая идентификация динамики: методы наименьших квадратов, возбуждающие траектории

## Часть VI. Кинематика мобильных роботов

### Кинематические модели колёсных роботов
- Дифференциальный привод (differential drive): управление скоростью двух колёс
  - Модель: v = (v_R + v_L)/2, ω = (v_R - v_L)/L
  - Кинематика: ẋ = v·cos(θ), ẏ = v·sin(θ), θ̇ = ω
  - ICR (Instantaneous Centre of Rotation): геометрический смысл поворота

- Роботы с омни-колёсами (omnidirectional):
  - 3-колёсный и 4-колёсный Mecanum/Swedish-wheel
  - Голономные vs неголономные системы — принципиальное различие!
  - Матрица колёсной скорости → скорость тела (прямая задача), и обратно

- Модель Аккермана (автомобиль, Ackermann steering):
  - Геометрия рулевого управления, угол поворота, радиус поворота
  - Кинематические уравнения bicycle model
  - Ограничение: минимальный радиус поворота, невозможность движения боком

### Одометрия и накопление ошибок
- Вычисление позиции по счётчикам энкодеров (одометрия)
- Источники ошибок: проскальзывание колёс, неровности поверхности, неточность геометрии
- Накопление ошибок одометрии: почему робот «дрейфует»
- Модели ошибок: Gaussian noise, velocity-based noise model

### Кинематика шагающих роботов (введение)
- Статическая vs динамическая устойчивость: criterion of support polygon
- Походки (gaits): tripod, wave, ripple gait для гексапода
- Кинематика одной ноги как последовательный манипулятор
- Синхронизация ног: центральные паттерн-генераторы (CPG)

## Часть VII. Планирование траекторий

### Планирование в пространстве сочленений (Joint Space)
- Полиномиальные траектории: кубический полином (4 ограничения: q0, qf, q̇0, q̇f)
- Полином пятой степени (6 ограничений: добавляются q̈0, q̈f)
- LSPB (Linear Segment with Parabolic Blend): трапециевидный профиль скорости
  - Фазы: разгон (парабола) → постоянная скорость (линейный) → торможение (парабола)
  - Расчёт времени перехода tb и скорости насыщения v_b
- Многосегментные траектории: сплайны, via-points

### Планирование в декартовом пространстве (Cartesian Space)
- Линейная траектория TCP (инструмента): интерполяция позиции p(t) = p0 + s(t)*(pf - p0)
- Интерполяция ориентации: SLERP (Spherical Linear Interpolation) для кватернионов
- Профиль скорости инструмента: приложение IK в каждой точке траектории
- Проблемы декартового планирования: сингулярности на пути, выход из workspace
- Joint-space vs Cartesian-space: trade-offs

### Алгоритмы планирования пути в конфигурационном пространстве
- Потенциальные поля (Potential Fields): притягивающий потенциал цели + отталкивающий потенциал препятствий
  - Градиентный спуск, проблема локальных минимумов
- RRT (Rapidly-exploring Random Trees): сэмплирование конфигурационного пространства
- RRT* и другие оптимальные планировщики
- PRM (Probabilistic Roadmap Method): offline precomputation + online query
- Практика: MoveIt! в ROS — библиотека планирования, OMPL

## Часть VIII. Параллельные механизмы и платформа Стюарта

### Параллельные манипуляторы: определение и преимущества
- Последовательные (serial) vs параллельные (parallel) манипуляторы — фундаментальное различие
- Преимущества параллельных: высокая жёсткость, точность, скорость, распределённая нагрузка
- Недостатки: меньший рабочий объём, сложная кинематика, проблема сингулярностей

### Платформа Стюарта (Stewart-Gough Platform)
- История: Д. Стюарт (1965) — оригинальная публикация для авиасимулятора
- Конфигурация 6-DOF платформы: 6 линейных приводов, соединяющих основание и платформу
- Обратная кинематика параллельного механизма: легко! (замкнутая форма)
  - Длина каждого привода: l_i = ||p + R*b_i - a_i||
- Прямая кинематика: сложно! (система 6 нелинейных уравнений, до 40 решений)
- Алгоритмы PKM FK: итерационные методы, нейросетевые приближения

### Дельта-робот
- Конфигурация: три параллельных кинематических цепочки, соединяющих основание и платформу
- Кинематика дельта-робота: геометрический анализ
- Рабочее пространство: форма «колокола», высокая скорость движения (pick-and-place)
- Применения: фармацевтика, пищевая промышленность, электроника

### Сингулярности параллельных механизмов
- Три типа сингулярностей параллельных механизмов (по Gosselin & Angeles):
  - Тип 1 (inverse): конфигурации, при которых привод не может создать усилие
  - Тип 2 (direct): конфигурации неограниченной подвижности платформы
  - Тип 3 (combined): обе одновременно
- Индекс манипулируемости параллельных механизмов

## Основоположники

### Пионеры кинематики роботов
- **Denavit & Hartenberg (1955)** — стандартное представление кинематических цепочек. Фундамент всей робототехники на 70 лет вперёд

- **Uicker (1965)** — первое применение Лагранжева формализма к манипуляторам
- **Luh, Walker, Paul (1980)** — рекурсивный алгоритм Ньютона-Эйлера, ставший стандартом для вычислений в реальном времени
- **Featherstone (1983, 2007)** — алгоритм сочленённого тела (Articulated Body Algorithm), O(n) сложность

# 5. ИНЖЕНЕРНЫЕ СВЯЗИ И ПРИМЕНЕНИЯ

## Связь с управлением
- FK/IK — основа вычисления управляющих сигналов: нельзя управлять роботом в декартовом пространстве без решения IK
- Динамическая модель τ = M(q)q̈ + C(q,q̇)q̇ + G(q) — фундамент для компенсации динамики в advanced-управлении

## Связь с планированием
- C-space концепция — основа всех алгоритмов планирования движений
- Якобиан — ключ к velocity-based планированию

## Связь с компьютерным зрением
- Калибровка Eye-in-Hand vs Eye-to-Hand: FK используется для вычисления позиции камеры
- Visual Servoing: якобиан изображения аналогичен геометрическому якобиану манипулятора

## Реальные системы
- **Universal Robots (UR5/UR10)**: сферическое запястье — аналитическая IK, открытый интерфейс для изучения
- **KUKA LBR iiwa**: 7DOF избыточный, torque-controlled — практика нуль-пространства
- **Boston Dynamics Spot**: 12 DOF ноги — параллельная кинематика + планирование походок

## Типичные ошибки студентов
1. Путаница порядка умножения матриц поворота (pre- vs post-multiplication)
2. Забывают о gimbal lock углов Эйлера при ориентации «прямо вверх»
3. Считают DH-параметры «неизменными» — забывают что θ_i и d_i могут быть переменными
4. Не понимают физический смысл якобиана — применяют формулы механически
5. Используют DLS-IK без анализа λ: слишком малый — нестабильность, слишком большой — медленность
