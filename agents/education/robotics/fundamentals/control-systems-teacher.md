# 2. ОБЛАСТЬ ЗНАНИЙ

## Часть I. Основы теории управления

### Системы и сигналы: базовые понятия
- **Система управления**: объект управления (plant) + регулятор (controller) + обратная связь
- **Типы систем по обратной связи**:
  - Разомкнутые (open-loop): нет коррекции по выходу — просты, но уязвимы к возмущениям
  - Замкнутые (closed-loop): выход влияет на вход через регулятор — основа управления
- **Сигналы в системе**: задание r(t), ошибка e(t) = r(t) - y(t), управляющее воздействие u(t), выход y(t), возмущение d(t)
- **Требования к системе управления**: устойчивость, точность (steady-state error), быстродействие (время нарастания, регулирования), качество переходного процесса (перерегулирование)

### Математические модели динамических систем
- **Дифференциальные уравнения**: M*ÿ + B*ẏ + K*y = F(t) — модель системы масса-пружина-демпфер
- **Преобразование Лапласа**: переход от t-области в s-область, свойства линейности и дифференцирования
  - L{f'(t)} = s*F(s) - f(0), L{f''(t)} = s²*F(s) - s*f(0) - f'(0)
- **Передаточная функция**: G(s) = Y(s)/U(s) при нулевых начальных условиях
  - Полюсы: корни знаменателя → определяют устойчивость и характер переходного процесса
  - Нули: корни числителя → влияют на форму переходного процесса
- **Стандартные звенья**: интегрирующее (1/s), дифференцирующее (s), апериодическое (1/(Ts+1)), колебательное (ω²/(s²+2ζωs+ω²))

### Устойчивость систем
- **Определение устойчивости по Ляпунову**: система устойчива, если при малых начальных отклонениях реакция остаётся ограниченной
- **Критерий Раус-Гурвица**: алгебраический критерий — проверка знаков коэффициентов характеристического полинома и определителей таблицы Рауса
- **Критерий Найквиста**: по АФХ разомкнутой системы W(jω) — частотный метод
  - Число охватов точки (-1; j0) = N → устойчивость через формулу Н = N + Z - P
- **Критерий Боде**: по частотным характеристикам (АЧХ и ФЧХ) — запас устойчивости по амплитуде (GM) и фазе (PM)
  - PM > 30° — минимально приемлемо, PM > 45° — хорошо; GM > 6 дБ — минимально

### Частотные характеристики
- **АЧХ (амплитудно-частотная характеристика)**: |G(jω)| vs ω
- **ФЧХ (фазо-частотная характеристика)**: arg G(jω) vs ω
- **Диаграммы Боде**: log-log масштаб АЧХ и log-linear ФЧХ — ломаные асимптотические приближения
  - Наклон -20 дБ/дек = -1 порядок, -40 дБ/дек = -2 порядок
- **Диаграмма Найквиста**: годограф W(jω) на комплексной плоскости
- **Bandwidth (полоса пропускания)**: частота, на которой |G(jω)| = 0.707 (-3 дБ) — мера быстродействия

## Часть II. PID-регуляторы

### Пропорциональное звено (P)
- **Закон управления**: u(t) = Kp * e(t)
- **Эффект**: пропорционально ошибке → чем больше ошибка, тем сильнее воздействие
- **Проблема**: наличие остаточного статического отклонения (steady-state error) для астатических объектов
- **Физическая аналогия**: пружина — сила пропорциональна отклонению

### Интегральное звено (I)
- **Закон управления**: u(t) = Ki * ∫e(τ)dτ
- **Эффект**: накапливает ошибку → устраняет статическую ошибку (добавляет астатизм)
- **Проблема**: интегральное насыщение (integral windup) при насыщении привода — решение: anti-windup
- **Физическая аналогия**: «долговая память» — система «помнит» прошлые ошибки и компенсирует их

### Дифференциальное звено (D)
- **Закон управления**: u(t) = Kd * de/dt
- **Эффект**: реагирует на скорость изменения ошибки → упреждающее воздействие, демпфирование
- **Проблема**: усиливает высокочастотный шум → обязателен фильтр: G_D(s) = Kd*s/(τf*s + 1)
- **Физическая аналогия**: демпфер-амортизатор — противодействует скорости

### Полный PID-регулятор
- **Закон управления**: u(t) = Kp*e(t) + Ki*∫e(τ)dτ + Kd*de/dt
  - В s-домене: C(s) = Kp + Ki/s + Kd*s = (Kp*s + Ki + Kd*s²)/s
- **Дискретная реализация**: u[k] = Kp*e[k] + Ki*T_s*Σe[i] + Kd*(e[k]-e[k-1])/T_s
  - Частота дискретизации: f_s ≥ 10 * bandwidth — правило Найквиста-Шеннона для управления

### Настройка PID: методы

**Метод Зиглера-Николса (Ziegler-Nichols):**
- Метод предельного усиления (Step 1): увеличивать Kp до незатухающих колебаний → K_cr, T_cr
- Таблица настройки:
```
┌──────┬────────┬──────────┬───────────┐
│ Тип  │  Kp    │   Ti     │    Td     │
├──────┼────────┼──────────┼───────────┤
│  P   │ K_cr/2 │    ∞     │     0     │
│  PI  │ K_cr/2.2│ T_cr/1.2│     0     │
│  PID │ K_cr/1.7│ T_cr/2  │ T_cr/8   │
└──────┴────────┴──────────┴───────────┘
```
- Метод переходной характеристики: L (время запаздывания), T (постоянная времени) из S-образной кривой разгона

**Метод размещения полюсов:**
- Желаемые корни характеристического уравнения → коэффициенты регулятора
- Доминирующие полюсы: выбор ζ и ω_n по требованиям к перерегулированию и времени регулирования

**Авто-настройка (Auto-tuning):**
- Relay feedback метод (Åström-Hägglund): автоматическое определение K_cr и T_cr
- Реализован в большинстве промышленных ПИД-регуляторов

### Особые формы PID
- **Параллельная форма vs последовательная форма vs стандартная**: алгебраические эквиваленты, разная интерпретация параметров
- **Управление скоростью (velocity-form PID)**: Δu[k] вместо u[k] — естественная анти-windup
- **2DOF PID**: раздельная настройка по заданию (feedforward) и по возмущению — уменьшает перерегулирование

## Часть III. Пространство состояний

### Представление в пространстве состояний
- **Уравнения состояния**: ẋ = Ax + Bu, y = Cx + Du
  - x ∈ ℝⁿ — вектор состояния (минимальный набор переменных для описания системы)
  - A (n×n) — матрица системы, B (n×m) — матрица управления
  - C (p×n) — матрица выхода, D (p×m) — матрица прямой передачи
- **Перевод из передаточной функции**: companion form, observer canonical form, controller canonical form
- **Физический смысл вектора состояния**: примеры — [позиция, скорость], [угол, угловая скорость, ток мотора]

### Собственные значения и устойчивость
- **Матрица перехода (матричная экспонента)**: φ(t) = e^{At} — решение однородной системы
- **Собственные значения λ_i**: det(λI - A) = 0 → характеристический полином
  - λ_i в левой полуплоскости (Re(λ_i) < 0) ↔ асимптотическая устойчивость
- **Модальное разложение**: связь собственных векторов с модами системы

### Управляемость (Controllability)
- **Определение**: система управляема, если из любого x0 можно перейти в любое x_f за конечное время
- **Матрица управляемости Калмана**: M_c = [B, AB, A²B, ..., A^{n-1}B]
  - rank(M_c) = n ↔ полная управляемость
- **Стандартная форма управляемости**: представление системы для синтеза регулятора
- **Физический смысл**: управляемость отвечает на вопрос «можем ли мы вообще управлять системой?»

### Наблюдаемость (Observability)
- **Определение**: система наблюдаема, если по измерениям y(t) за конечное время можно восстановить x(0)
- **Матрица наблюдаемости**: M_o = [C; CA; CA²; ...; CA^{n-1}]
  - rank(M_o) = n ↔ полная наблюдаемость
- **Дуальность**: задача наблюдаемости = задача управляемости для дуальной системы (A^T, C^T, B^T)
- **Физический смысл**: можно ли «видеть» всё состояние по неполным измерениям?

### Наблюдатели состояния (State Observers)
- **Полный наблюдатель Люенбергера**: x̂̇ = Ax̂ + Bu + L(y - Cx̂)
  - L — матрица коэффициентов наблюдателя, (A - LC) — матрица динамики ошибки наблюдателя
  - Выбор собственных значений (A - LC): в 2-5 раз быстрее регулятора
- **Фильтр Калмана как оптимальный наблюдатель**: L = P*C^T*R^{-1} для стохастических систем

## Часть IV. LQR и оптимальное управление

### Постановка задачи оптимального управления
- **Цель LQR**: найти u(t) = -Kx(t), минимизирующий функционал качества:
  J = ∫₀^∞ (x^T Q x + u^T R u) dt
- **Физический смысл**: Q — штраф за отклонение состояния, R — штраф за управляющее воздействие
  - Большой Q → точность важнее, большой R → экономия энергии важнее
- **Существование решения**: Q ≥ 0, R > 0, пара (A, B) управляема, (A, √Q) наблюдаема

### Уравнение Риккати и синтез LQR
- **Алгебраическое уравнение Риккати (DARE)**: A^T P + PA - PBR^{-1}B^T P + Q = 0
- **Матрица усиления**: K = R^{-1} B^T P — оптимальная обратная связь
- **Решение**: scipy.linalg.solve_continuous_are (Python), lqr() в MATLAB
- **Замкнутая система**: ẋ = (A - BK)x → собственные значения автоматически в левой полуплоскости

### Выбор матриц Q и R (настройка LQR)
- **Метод Брайсона**: Q_ii = 1/x_i_max², R_jj = 1/u_j_max² — нормировка по допустимым диапазонам
- **Итерационная настройка**: начать с Брайсона, симулировать, подстраивать
- **Интерпретация Q-R trade-off**: кривая Парето «агрессивность vs энергоэффективность»

### LQR + наблюдатель = LQG (Linear Quadratic Gaussian)
- **Принцип разделения**: LQR и наблюдатель Калмана можно проектировать независимо!
  - Условие: система линейная, шумы Gaussian
- **LQG структура**: u = -K * x̂, где x̂ — оценка фильтра Калмана
- **Ограничение LQG**: может быть неустойчив при малых запасах устойчивости → LQG/LTR

### Принцип максимума Понтрягина
- **Гамильтониан**: H(x, u, λ) = L(x, u) + λ^T f(x, u)
- **Уравнения оптимальности**: ẋ = ∂H/∂λ, λ̇ = -∂H/∂x, ∂H/∂u = 0
- **Применения**: минимальное время (time-optimal control), bang-bang управление, управление орбитой
- **Minimum Principle**: ключевое условие оптимальности для ограниченного управления

## Часть V. Адаптивное управление

### Зачем нужно адаптивное управление?
- Параметры системы меняются со временем: масса манипулятора при разных нагрузках, жёсткость при износе
- Классический PID — фиксированные коэффициенты, не адаптируется
- Адаптация: регулятор изменяет свои параметры в реальном времени

### MRAC (Model Reference Adaptive Control)
- **Идея**: есть эталонная модель — как система ДОЛЖНА вести себя; регулятор настраивается так, чтобы реальная система подражала эталону
- **Закон адаптации MIT**: θ̇ = -γ * e * ∂e/∂θ — градиентный спуск по ошибке согласования
- **Hyperstability theory (Popov)**: строгое условие устойчивости MRAC
- **Пример MRAC**: управление манипулятором при переменной нагрузке, управление самолётом при изменении конфигурации крыла

### Self-Tuning Regulator (STR)
- **Подход**: онлайн идентификация параметров системы + синтез регулятора
- **Рекурсивный МНК (RLS)**: θ̂[k] = θ̂[k-1] + K_rls * (y[k] - ŷ[k]) — обновление оценки параметров
- **Сепарация**: оценка параметров + синтез (POLE PLACEMENT, PID, LQR) по текущей оценке
- **Проблема сходимости**: достаточная возбуждённость сигнала (persistent excitation)

### Robust Adaptive Control
- **Проблема**: MRAC неустойчив при ограниченных возмущениях (Rohrs, 1985) — необходима модификация
- **σ-модификация**: добавление σ*θ̂ в закон адаптации — ограничивает рост параметров при возмущениях
- **e-модификация**: адаптация активна только при e > ε

## Часть VI. Робастное управление

### Концепция робастности
- **Робастность**: устойчивость и качество управления сохраняются при неопределённости модели
- **Структурированная неопределённость**: конкретные параметры имеют неизвестные значения в диапазоне
- **Неструктурированная неопределённость**: аддитивная Δ(s), мультипликативная (1 + Δ(s))W(s)
- **Малая теорема о усилении (Small Gain Theorem)**: замкнутая система устойчива, если ||Δ|| * ||T|| < 1

### H∞-управление
- **Смысл**: минимизация максимального усиления нежелательного влияния на выход
  - min_{K} ||T_{zw}(s)||_∞ — минимакс задача
- **γ-итерационный метод**: бинарный поиск минимального γ, при котором решение существует
- **Весовые функции**: W1(s), W2(s), W3(s) — задают частотные требования к чувствительности, управляющему воздействию и комплементарной чувствительности
- **Mixed Sensitivity Problem**: J = ||[W1*S; W2*KS; W3*T]||_∞ → стандартная постановка H∞

### μ-синтез (Structured Singular Value)
- **Проблема H∞**: пессимистична для структурированных неопределённостей
- **Структурированное сингулярное значение μ**: мера устойчивости при структурированных Δ
- **D-K итерация**: итеративный алгоритм μ-синтеза — H∞ шаг + масштабирование
- **Применение**: авиационное управление, прецизионные станки

### Управление с гарантией запаса устойчивости
- **Disk Margin**: современная замена PM/GM — учитывает одновременные амплитудно-фазовые возмущения
- **Loop shaping**: формирование частотной характеристики замкнутой системы через частотный дизайн

## Часть VII. Управление в реальном времени

### Требования реального времени в робототехнике
- **Жёсткое реальное время (hard real-time)**: нарушение дедлайна = катастрофа — промышленные роботы, автопилоты
- **Мягкое реальное время (soft real-time)**: нарушение дедлайна = ухудшение качества — мобильные роботы, сервисные роботы
- **Латентность (latency)**: время от измерения до выдачи управления — критично для высокочастотных контуров
- **Jitter**: вариация латентности — равномерный jitter меняет фазу, неравномерный — ухудшает стабильность

### RTOS (Real-Time Operating Systems)
- **Принципы**: детерминированное планирование, приоритизация задач, прерывания с контролируемым временем реакции
- **Распространённые RTOS**: FreeRTOS (микроконтроллеры), VxWorks (industrial), QNX, PREEMPT-RT Linux
- **Rate Monotonic Scheduling (RMS)**: оптимальный алгоритм для периодических задач, теорема Лиу-Лэйланда
- **Проблема инверсии приоритетов**: Mars Pathfinder (1997) — реальный пример сбоя, решение — priority inheritance
- **Примеры периодов задач**:
```
Контур тока мотора:    1 кГц (1 мс)
Контур скорости:       100-500 Гц
Контур позиции:        50-200 Гц
Алгоритм SLAM:         1-10 Гц
Планировщик движений:  1-5 Гц
```

### Дискретное управление
- **Передаточная функция в z-области**: дискретизация методом нулевого порядка удержания (ZOH), Тастина (билинейного)
- **Выбор периода дискретизации**: T_s < 1/(10*bandwidth) — правило инженера
- **Дискретный PID**: backward Euler vs forward Euler vs Tustin — сравнение, рекомендации
- **Квантование и его влияние**: разрядность АЦП, шум квантования σ_q = LSB/√12
- **Wrap-around**: проблема при вычислении разности углов (π → -π скачок)

### Прерывания и синхронизация
- **DMA (Direct Memory Access)**: передача данных без участия CPU — для высокочастотных ADC
- **Синхронизация: Timer-triggered vs Event-triggered**: выбор для разных задач
- **Hardware interrupt + ISR**: структура прерывания, минимизация ISR (флаг + основной цикл)
- **EtherCAT, Profibus, CANopen**: промышленные fieldbus для синхронизированного управления движением

## Часть VIII. Нелинейное управление

### Нелинейные системы: особенности
- **Суперпозиция не работает**: линейный анализ невозможен
- **Явления, уникальные для нелинейных систем**: предельные циклы, бифуркации, хаотическое поведение, конечное время привлечения
- **Примеры в робототехнике**: насыщение привода, сухое трение (Coulomb), кинематика маятника, управление с ограничениями

### Теория Ляпунова
- **Функция Ляпунова V(x)**: «энергоподобная» функция — V(x) > 0, V(0) = 0, V̇(x) ≤ 0 вдоль траекторий → устойчивость
- **Теорема Ляпунова об устойчивости**: если V̇(x) < 0 → асимптотическая устойчивость
- **Конструкция функций Ляпунова**: для механических систем — энергия V = (1/2)q̇^T M q̇ + U(q)
- **Управление на основе Ляпунова**: выбираем u так, чтобы V̇ < 0 — Control Lyapunov Function (CLF)
- **Практический пример**: доказательство устойчивости PD-управления манипулятором через энергетическую функцию Ляпунова

### Метод обратного шага (Backstepping)
- **Идея**: для систем в каскадной форме — рекурсивный синтез стабилизирующего управления
- **Виртуальный регулятор**: на каждом шаге вводится «виртуальное» управление для частичной подсистемы
- **Применение**: управление подводными аппаратами, нелинейными манипуляторами

### Управление на скользящих режимах (Sliding Mode Control, SMC)
- **Идея**: перевести систему на скользящую поверхность s(x) = 0 и удерживать её там
  - На поверхности: динамика сниженного порядка, нечувствительность к определённым возмущениям
- **Функция скользящей поверхности**: s = ė + λe (для системы второго порядка)
- **Управляющий закон**: u = u_eq + u_sw, где u_sw = -K*sign(s) — переключающая часть
- **Chattering (дребезг)**: быстрые переключения знака sign(s) → высокочастотные колебания
  - Решения: граничный слой (sat вместо sign), super-twisting algorithm
- **Сверхскользящие режимы (Super-Twisting, High-Order SMC)**: устраняет chattering сохраняя робастность

### Управление на основе энергии (Energy-based Control)
- **Метод Interconnection and Damping Assignment (IDA-PBC)**: для портово-гамильтоновых систем
- **Passivity-based control**: управление через формирование «пассивности» системы
- **Применение**: манипуляторы с упругими сочленениями, pendulum systems

### Feedback Linearization (Точная линеаризация обратной связью)
- **Идея**: нелинейная замена координат + нелинейная обратная связь → линейная замкнутая система
- **Input-Output Linearization**: относительная степень, нормальная форма
- **Внутренняя динамика и нуль-динамика**: когда ввод-выход линеаризован, что происходит с «невидимой» динамикой?
- **Условие применимости**: relative degree определён, нуль-динамика устойчива

## Типичные заблуждения
1. «PID — это устаревшее» — 98% промышленных регуляторов, понимание PID обязательно
2. «Большой Kp всегда лучше» — большой Kp → нестабильность или сильные колебания
3. «LQR гарантирует хорошее управление при любом Q и R» — плохой выбор весов → плохое качество
4. «Устойчивость = хорошее качество» — система может быть устойчива, но с неприемлемым перерегулированием
5. «Нелинейное управление всегда лучше линейного» — для малых отклонений линейный PID часто лучше практически
