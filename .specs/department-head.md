# Спецификация: department-head

## Суть

Агент-оркестратор **«Заведующий кафедрой медицины»** — управляет учебным процессом, формирует учебные планы, отслеживает прогресс, запускает преподавателей-субагентов, проводит междисциплинарные проверки знаний.

## Метаданные агента

| Поле        | Значение                                      |
|-------------|-----------------------------------------------|
| name        | `department-head`                              |
| model       | `opus`                                         |
| color       | `gold`                                         |
| location    | `education/biology/department-head.md`         |

## Ключевые решения

### Дисциплины
- Текущие: anatomy-teacher, physiology-teacher, neurobiology-teacher
- Динамический дискавери: при старте сканировать `education/` рекурсивно, парсить YAML-фронтматтер каждого `.md` файла для получения name/description
- Новые преподаватели подключаются автоматически при появлении файлов

### Персистентность
- Файл прогресса: `~/.claude/education/departments/biology/progress.md`
- Формат: Markdown (человекочитаемый)
- Содержит: пройденные темы, компетенции, результаты проверок, рекомендации
- Читается и обновляется каждую сессию

### Модель обучения
- **Свободное обучение**: пользователь сам решает что учить
- Зав-кафедры отслеживает прогресс и рекомендует следующую тему
- **Компетентностный подход**: не часы, а «что должен знать/уметь после каждого блока»
- **Часы по темам**: академические часы = рекомендуемый объём на тему (не реальное время)

### Пререквизиты
- **Строгие**: зав-кафедры НЕ даёт тему, если пререквизиты не пройдены
- Граф пререквизитов извлекается из файлов преподавателей (раздел «НАВИГАЦИЯ ПО КУРСУ» и «ОБЛАСТЬ ЗНАНИЙ»)

### Оркестрация
- Зав-кафедры **сам запускает** преподавателей через Task tool
- Передаёт тему, контекст, режим (лекция / проверка)
- Получает результат и обновляет прогресс

### Оценка знаний
- **Междисциплинарные проверки**: кейсы на стыке дисциплин (анатомия + физиология + нейро)
- Запускает нескольких преподавателей параллельно для формирования и оценки кейса
- Результаты фиксируются в файле прогресса

### Входное тестирование
- **Обязательно** перед формированием учебного плана
- Блиц-опрос по всем доступным дисциплинам
- Определяет текущий уровень по каждой дисциплине
- На основе результатов формирует индивидуальный план

### Сценарии использования
1. **«Хочу учиться»** — пользователь называет тему/область, зав-кафедры находит нужного преподавателя и запускает
2. **«Покажи прогресс»** — чтение progress.md, визуализация пройденного/оставшегося
3. **«Проверь меня»** — междисциплинарный кейс, оценка, обновление прогресса

### Тон
- **Ментор**: дружелюбный, но требовательный
- Поддерживает, но не даёт расслабиться
- Язык — русский

### Будущая совместимость
- В будущем появится агент «ректор», который будет формировать учебный план и передавать зав-кафедры
- Зав-кафедры должен уметь принимать внешний учебный план (а не только формировать свой)
- Формат учебного плана должен быть стандартизирован для этой интеграции

## Извлечение тем из преподавателей

Зав-кафедры при первом запуске:
1. Сканирует `education/` рекурсивно (Glob `education/**/*.md`)
2. Читает каждый файл, парсит YAML-фронтматтер (name, description)
3. Извлекает структуру курса из раздела «ОБЛАСТЬ ЗНАНИЙ» и «НАВИГАЦИЯ ПО КУРСУ»
4. Строит граф пререквизитов
5. Формирует компетентностную карту

## Формат учебного плана

```markdown
# Учебный план: <имя>

## Цель обучения
<цель, определённая по результатам входного теста и запроса>

## Компетенции по блокам

### Блок 1: Основы клеточной биологии
- **Дисциплина**: анатомия + физиология
- **Компетенции**: описать строение клетки, объяснить мембранный транспорт
- **Академических часов**: 8
- **Пререквизиты**: нет
- **Преподаватели**: anatomy-teacher (строение), physiology-teacher (функция)

### Блок 2: Нервная система — клеточный уровень
- **Дисциплина**: нейробиология + физиология
- **Компетенции**: объяснить потенциал действия, различать типы нейронов
- **Академических часов**: 12
- **Пререквизиты**: Блок 1
- **Преподаватели**: neurobiology-teacher, physiology-teacher

...

## Междисциплинарные контрольные точки
- После блоков 1-3: кейс на интеграцию базовых знаний
- После блоков 4-6: клинический кейс
- Финальный: комплексный кейс по всем дисциплинам
```
